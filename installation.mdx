    **Installation Steps:**

    <Steps>
      <Step title="Download the DMG file">
        Click the download link above for your Mac's processor type.
      </Step>
      <Step title="Open the DMG file">
        Double-click the downloaded `.dmg` file to mount it.
      </Step>
      <Step title="Drag to Applications">

      </Step>

    </Note>

    ### Homebrew Installation

    For developers who prefer package managers:

    ```bash
    brew install --cask conar
    ```

    **Update via Homebrew:**

    ```bash
    brew upgrade conar
    ```

    ### System Requirements

    - macOS 10.13 (High Sierra) or later
    - 200 MB free disk space
    - Internet connection for cloud sync and AI features

    ### Built With ToDesktop

    The macOS app is built using ToDesktop with the following configuration:

    ```json
    {
      "appId": "com.wannabespace.conar",
      "category": "public.app-category.developer-tools",
      "appProtocolScheme": "conar"
    }
    ```

    This ensures native macOS integration including:
    - App protocol handlers (`conar://`)
    - Native notifications
    - System tray integration
    - Auto-updates
  </Tab>

    ### Debian/Ubuntu Installation

    ```bash
    # Download the .deb package
    wget https://download.conar.app/linux/deb/x64 -O conar.deb

    # Install the package
    sudo dpkg -i conar.deb

    # Install dependencies if needed
    sudo apt-get install -f

    # Launch Conar
    conar
    ```

    ### AppImage Installation

    ```bash
    # Download AppImage
    wget https://download.conar.app/linux/appImage/x64 -O Conar.AppImage

    # Make it executable
    chmod +x Conar.AppImage

    # Run the app
    ./Conar.AppImage
    ```

    <Tip>
      You can move the AppImage to `~/.local/bin/` or `/usr/local/bin/` to run it from anywhere.
    </Tip>

    ### RPM Installation (Fedora/RHEL)

    ```bash
    # Download the .rpm package
    wget https://download.conar.app/linux/rpm/x64 -O conar.rpm

    # Install the package
    sudo rpm -i conar.rpm

    # Or using dnf
    sudo dnf install ./conar.rpm

    # Launch Conar
    conar
    ```

    ### Arch Linux (AUR)

    For Arch Linux users, Conar is available on the AUR:

    ```bash
    # Using yay
    yay -S conar-bin

    # Or using paru
    paru -S conar-bin
    ```

    **AUR Package Details:**

    ```bash
    Package: conar-bin
    Description: AI-powered tool for working with Postgres, MySQL, MSSQL and ClickHouse
    License: AGPL-3.0-or-later
    Maintainer: Bouteiller a2n Alan <hi@a2n.dev>
    ```

    ### System Requirements

    - 64-bit Linux distribution
    - X11 or Wayland display server
    - 200 MB free disk space
    - Internet connection for cloud sync and AI features

    **Dependencies (automatically installed):**

    The Debian package includes these dependencies:

    ```text
    alsa-lib, at-spi2-core, cairo, dbus, expat, gcc-libs,
    glib2, glibc, gtk3, hicolor-icon-theme, libcups,
    libx11, libxcb, libxcomposite, libxdamage, libxext,
    libxfixes, libxkbcommon, libxrandr, mesa, nspr,
    nss, pango, systemd-libs
    ```

    ### Desktop Integration

    After installation, Conar will appear in your application menu under the "Development" category.

    ```bash
    # Command line usage
    conar

    # Or use the application launcher
    # The .desktop file is installed to:
    /usr/share/applications/conar.desktop
    ```

    ### Troubleshooting

    If you encounter issues launching the app:

    ```bash
    # Check if installed correctly
    which conar

    # Run with verbose output
    conar --verbose

    # Check system logs
    journalctl -xe | grep conar
    ```

### Progressive Web App (PWA)

You can install the web app as a PWA for a more app-like experience:

<Steps>

    Navigate to [https://conar.app](https://conar.app)
  </Step>
  <Step title="Install the PWA">

    - **Safari**: Click Share → Add to Home Screen
    - **Firefox**: Look for the install prompt
  </Step>
  <Step title="Launch from your home screen">
    The app will now appear as a standalone application
  </Step>
</Steps>
