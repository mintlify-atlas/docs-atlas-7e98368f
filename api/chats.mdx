## Overview

The Chats API manages AI-powered chat sessions for generating and refining SQL queries. Each chat is associated with a database connection and contains a conversation history.

<Note>
All chat endpoints require an active Pro subscription.
</Note>

## Endpoints

### Create Chat

Create a new AI chat session.

<CodeGroup>
```bash Request
POST /rpc/chats.create
Content-Type: application/json

{
  "connectionId": "01932e8f-5f5e-7b8a-9f3d-4c5e6a7b8c9d"
}
```

```json Response
{}
```
</CodeGroup>

<ParamField path="connectionId" type="string" required>
  UUID of the database connection for this chat
</ParamField>

<Note>
The `databaseId` parameter is deprecated but still supported. Use `connectionId` instead.
</Note>

### Update Chat

Update chat properties like title.

<CodeGroup>
```bash Request
POST /rpc/chats.update
Content-Type: application/json

{
  "id": "01932e8f-5f5e-7b8a-9f3d-4c5e6a7b8c9d",
  "title": "Monthly Sales Report"
}
```

```json Response
{}
```
</CodeGroup>

<ParamField path="id" type="string" required>
  UUID of the chat to update
</ParamField>

<ParamField path="title" type="string">
  Updated chat title
</ParamField>

<Note>
You cannot update `userId`, `connectionId`, `activeStreamId`, `createdAt`, or `updatedAt` fields.
</Note>

### Remove Chat

<CodeGroup>
```bash Single Chat

Content-Type: application/json
```

## Data Model

### Chat Object

```typescript
interface Chat {
  id: string              // UUIDv7
  userId: string          // UUIDv7, foreign key to users
  connectionId: string    // UUIDv7, foreign key to connections
  title: string | null    // Auto-generated title
  activeStreamId: string | null // Current streaming message ID
  createdAt: Date
  updatedAt: Date
  messages?: ChatMessage[] // Included in list response
}
```

### Chat Message Object

```typescript
interface ChatMessage {
  id: string              // UUIDv7
  chatId: string          // UUIDv7, foreign key to chats
  role: 'user' | 'assistant' | 'system'
  parts: MessagePart[]    // Encrypted message parts
  metadata?: object       // Encrypted metadata
  createdAt: Date
  updatedAt: Date
}

interface MessagePart {
  type: 'text' | 'tool-call' | 'tool-result'
  text?: string
  toolCallId?: string
  toolName?: string
  args?: object
  result?: unknown
}
```

## Message Management

For creating and managing individual chat messages, see the Chat Messages API:

- `POST /rpc/chatsMessages.create` - Add messages to a chat
- `POST /rpc/chatsMessages.update` - Update message content
- `POST /rpc/chatsMessages.remove` - Delete messages
- `POST /rpc/chatsMessages.sync` - Sync messages

## Usage Example

```javascript
// 1. Create a new chat
await client.chats.create({
  connectionId: "01932e8f-5f5e-7b8a-9f3d-4c5e6a7b8c9d"
})

// 2. List all chats with messages
const chats = await client.chats.list()

// 3. Update chat title
await client.chats.update({
  id: "01932e8f-5f5e-7b8a-9f3d-4c5e6a7b8c9d",
  title: "Sales Analysis"
})

// 4. Remove a chat
await client.chats.remove({
  id: "01932e8f-5f5e-7b8a-9f3d-4c5e6a7b8c9d"
})
```
