## Overview

The AI API provides intelligent features for generating, fixing, and enhancing SQL queries using state-of-the-art language models.

<Note>
All AI endpoints require an active Pro subscription.
</Note>

## Endpoints

### Chat

Stream AI-generated SQL queries and responses in real-time.

<CodeGroup>
```bash Request
POST /rpc/ai.chat
Content-Type: application/json

{
  "id": "01932e8f-5f5e-7b8a-9f3d-4c5e6a7b8c9d",
  "type": "postgres",
  "context": "Database schema: users(id, name, email, created_at), orders(id, user_id, amount, created_at)",
  "createdAt": "2024-01-15T10:30:00.000Z",
  "updatedAt": "2024-01-15T10:30:00.000Z",
  "messages": [
    {
      "id": "01932f00-1111-2222-3333-444444444444",
      "role": "user",
      "parts": [
        {
          "type": "text",
          "text": "Show me total sales by month for the last year"
        }
      ]
    }
  ]
}
```

```text Response (Server-Sent Events)
event: message
data: {"type":"text-delta","text":"Here"}

event: message
data: {"type":"text-delta","text":"'s"}

event: message  
data: {"type":"text-delta","text":" a"}

event: message
data: {"type":"text-delta","text":" query"}

...

event: message
data: {"type":"finish","finishReason":"stop"}
```
</CodeGroup>

<ParamField path="id" type="string" required>
  UUIDv7 of the chat session
</ParamField>

<ParamField path="type" type="enum" required>
  Database type (e.g., `postgres`, `mysql`, `sqlite`)
</ParamField>

<ParamField path="context" type="string" required>
  Database schema and context information
</ParamField>

<ParamField path="messages" type="array" required>
  Array of chat messages with user prompts and AI responses
</ParamField>

<ParamField path="createdAt" type="string" required>
  ISO 8601 timestamp of chat creation
</ParamField>

<ParamField path="updatedAt" type="string" required>
  ISO 8601 timestamp of last update
</ParamField>

<ResponseField name="Stream Events" type="Server-Sent Events">
  The endpoint streams responses using Server-Sent Events (SSE) format:
  - `text-delta` - Incremental text chunks
  - `tool-call` - AI tool invocations
  - `tool-result` - Results from tool calls
  - `finish` - Stream completion with finish reason
</ResponseField>

<Note>
The chat endpoint uses Claude Sonnet 4.5 with fallback to Claude Opus 4.5 and Gemini 2.5 Pro for high availability.
</Note>

### Enhance Prompt

Improve and clarify user prompts for better SQL generation.

<CodeGroup>
```bash Request
POST /rpc/ai.enhancePrompt
Content-Type: application/json

{
  "prompt": "show sales",
  "chatId": "01932e8f-5f5e-7b8a-9f3d-4c5e6a7b8c9d"
}
```

```json Response
"Show total sales amount grouped by month for the last 12 months, ordered by date descending"
```
</CodeGroup>

<ParamField path="prompt" type="string" required>
  User's original prompt
</ParamField>

<ParamField path="chatId" type="string" required>
  UUIDv7 of the chat session for context
</ParamField>

<ResponseField name="text" type="string">
  Enhanced, clarified prompt
</ResponseField>

### Fix SQL

Automatically fix SQL syntax errors.

<CodeGroup>
```bash Request
POST /rpc/ai.fixSQL
Content-Type: application/json

{
  "sql": "SELECT * FROM users WHERE created_at > '2024-01-01",
  "error": "ERROR: unterminated string literal",
  "type": "postgres"
}
```

```json Response
"SELECT * FROM users WHERE created_at > '2024-01-01'"
```
</CodeGroup>

<ParamField path="sql" type="string" required>
  SQL query with error
</ParamField>

<ParamField path="error" type="string" required>
  Error message from the database
</ParamField>

<ParamField path="type" type="enum" required>
  Database type
</ParamField>

<ResponseField name="text" type="string">
  Fixed SQL query
</ResponseField>

### Update SQL

Modify existing SQL based on natural language instructions.

<CodeGroup>
```bash Request
POST /rpc/ai.updateSQL
Content-Type: application/json

{
  "sql": "SELECT * FROM users",
  "prompt": "add a WHERE clause to filter active users only",
  "type": "postgres",
  "context": "users table has a status column with values: active, inactive, suspended"
}
```

```json Response
"SELECT * FROM users WHERE status = 'active'"
```
</CodeGroup>

<ParamField path="sql" type="string" required>
  Original SQL query
</ParamField>

<ParamField path="prompt" type="string" required>
  Natural language instruction for modification
</ParamField>

<ParamField path="type" type="enum" required>
  Database type
</ParamField>

<ParamField path="context" type="string" required>
  Database schema context
</ParamField>

<ResponseField name="text" type="string">
  Updated SQL query
</ResponseField>

### Generate Title

Auto-generate a descriptive title for a chat session.

<CodeGroup>
```bash Request
POST /rpc/ai.generateTitle
Content-Type: application/json

{
  "chatId": "01932e8f-5f5e-7b8a-9f3d-4c5e6a7b8c9d",
  "messages": [
    {
      "id": "msg1",
      "role": "user",
      "parts": [
        {"type": "text", "text": "Show total sales by month"}
      ]
    }
  ]
}
```

```json Response
"Monthly Sales Analysis"
```
</CodeGroup>

<ParamField path="chatId" type="string" required>
  UUIDv7 of the chat session
</ParamField>

<ParamField path="messages" type="array" required>
  Chat messages to analyze for title generation
</ParamField>

<ResponseField name="text" type="string">
  Generated title (max 30 characters, title case)
</ResponseField>

<Note>
The title is automatically saved to the chat record in the database.
</Note>

### Generate Filters

Convert natural language to structured database filters.

<CodeGroup>
```bash Request
POST /rpc/ai.filters
Content-Type: application/json

{
  "prompt": "users created in the last 7 days with email containing gmail",
  "context": "users table: id, name, email (string), created_at (timestamp), status (enum: active, inactive)"
}
```

```json Response
{
  "filters": [
    {
      "column": "created_at",
      "operator": ">=",
      "values": ["2024-01-08T00:00:00.000Z"]
    },
    {
      "column": "email",
      "operator": "contains",
      "values": ["gmail"]
    }
  ]
}
```
</CodeGroup>

<ParamField path="prompt" type="string" required>
  Natural language filter description
</ParamField>

<ParamField path="context" type="string" required>
  Table schema with column names and types
</ParamField>

<ResponseField name="filters" type="array">
  Array of structured filter objects
</ResponseField>

<ResponseField name="filters[].column" type="string">
  Column name to filter on
</ResponseField>

<ResponseField name="filters[].operator" type="enum">
  Filter operator (e.g., `=`, `>`, `<`, `>=`, `<=`, `contains`, `startsWith`, `endsWith`)
</ResponseField>

<ResponseField name="filters[].values" type="array">
  Array of filter values
</ResponseField>

## AI Tools

The chat endpoint has access to specialized tools:

- **Database Query Tool** - Execute SELECT queries to fetch data
- **Web Search Tool** - Search for external information and documentation
- **Context7 MCP** - Access library/API documentation and setup guides

## System Prompts

The AI chat uses carefully crafted system prompts:

- Generates valid SQL for the specific database type
- Uses 2-space indentation and consistent formatting
- Quotes table/column names to prevent case sensitivity errors
- Provides SQL in markdown code blocks
- Responds in the same language as the user's message
- Considers performance implications
- Uses current date/time context

## Streaming Response Format

The chat endpoint uses Server-Sent Events (SSE) with the following event types:

```typescript
type StreamEvent = 
  | { type: 'text-delta', text: string }
  | { type: 'tool-call', toolCallId: string, toolName: string, args: object }
  | { type: 'tool-result', toolCallId: string, result: unknown }
  | { type: 'finish', finishReason: 'stop' | 'length' | 'error' }
```

## Usage Example

```javascript
// 1. Enhance user prompt
const enhancedPrompt = await client.ai.enhancePrompt({
  prompt: "show sales",
  chatId: chatId
})

// 2. Stream AI chat response
const stream = await client.ai.chat({
  id: chatId,
  type: "postgres",
  context: schemaContext,
  messages: [...messages, { role: 'user', parts: [{ type: 'text', text: enhancedPrompt }] }],
  createdAt: new Date(),
  updatedAt: new Date()
})

// 3. Fix SQL error
const fixedSQL = await client.ai.fixSQL({
  sql: "SELECT * FROM users WHERE",
  error: "ERROR: syntax error at end of input",
  type: "postgres"
})

// 4. Generate filters
const filters = await client.ai.filters({
  prompt: "active users from last week",
  context: "users(id, status, created_at)"
})
```
