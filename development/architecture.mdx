Conar is built as a monorepo containing multiple applications and shared packages that work together to provide a seamless database management experience.

## Monorepo Structure

The project uses **pnpm workspaces** for monorepo management:

```yaml
# pnpm-workspace.yaml
packages:
  - packages/*
  - apps/*
```

This structure enables:
- **Code sharing** between apps and packages
- **Consistent dependencies** via workspace catalog
- **Parallel builds** with Turbo
- **Type safety** across boundaries

## Data Flow

Here's how data flows through the system:

```mermaid
graph TD
    A[User Interface] -->|Action| B[React Query]
    B -->|Request| C[oRPC Client]
    C -->|HTTP| D[Hono API]
    D -->|Decrypt| E[@conar/shared]
    E -->|Parse| F[@conar/connection]
    F -->|Query| G[Database]
    G -->|Results| D
    D -->|Encrypt| E
    E -->|Response| C
    C -->|Cache| B
    B -->|Render| H[@conar/table]
    H -->|Display| A
```

### Connection Flow

1. **User Input**: User enters connection string
2. **Encryption**: Connection string encrypted with user password
3. **Storage**: Encrypted string stored in database (API) or electron-store (Desktop)
4. **Decryption**: On use, string decrypted with password
5. **Parsing**: `parseConnectionString()` extracts connection details
6. **Connection**: Database driver establishes connection

### Query Flow

1. **Editor**: User writes SQL in Monaco editor
2. **Execution**: Query sent to API via oRPC
3. **Connection Pool**: API uses pooled connection
4. **Database**: Query executed against database
5. **Results**: Raw results returned
6. **Rendering**: `@conar/table` virtualizes large result sets
7. **Display**: User sees results in virtual table

## Build System

### Turbo Configuration

Turbo orchestrates builds across the monorepo:

```json
{
  "pipeline": {
    "dev": {
      "dependsOn": ["^build"],
      "cache": false
    },
    "build": {
      "dependsOn": ["^build"],
      "outputs": ["dist/**", ".next/**"]
    },
    "test:unit": {
      "dependsOn": ["^build"]
    }
  }
}
```

### TypeScript Configuration

Project references ensure type safety:

```json
{
  "references": [
    { "path": "./packages/shared" },
    { "path": "./packages/connection" },
    { "path": "./packages/ui" },
    { "path": "./packages/table" }
  ]
}
```

## Performance Optimizations

### Virtual Scrolling

`@conar/table` uses TanStack Virtual to render only visible rows:

```typescript
const { getVirtualItems: getVirtualRows } = useVirtualizer({
  count: rows.length,
  getScrollElement: () => scrollRef.current,
  estimateSize: () => estimatedRowSize,
  overscan: 10, // Render 10 extra rows for smooth scrolling
})
```

### React Compiler

Both web and desktop apps use the experimental React Compiler:

```json
{
  "plugins": [["babel-plugin-react-compiler", {}]]
}
```

Vite automatically code-splits routes and heavy dependencies.
